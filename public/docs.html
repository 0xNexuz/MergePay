MERGEPAY DOCS

From merged code to an auditable onchain payout.

MergePay is an autonomous settlement agent for open-source bounties. It authenticates GitHub events, applies maintainer-defined treasury rules, and delegates reliable USDC execution to KeeperHub.

Production safety

Live mode stays locked until the KeeperHub key, token, webhook secret, admin token and durable audit store all pass readiness checks.

How it works

  1. Maintainer authorizesAdd bounty-approved and a label such as bounty-usdc:25.
  2. Contributor identifies the recipientAdd mergepay-recipient: 0x... to the pull-request description.
  3. GitHub proves the mergeA signed pull-request webhook reaches MergePay.
  4. MergePay decidesSignature, repository, labels, address, ceiling and replay rules are checked.
  5. KeeperHub executesThe transfer is simulated, submitted idempotently and checked for a confirmed hash.
  6. MergePay records proofThe decision and explorer receipt appear in the audit panel.

Payout policy

The contributor controls only the recipient address. The maintainer controls authorization and amount through GitHub labels. MergePay also enforces an allowlisted repository and global USD ceiling.

Security model

Authenticated events

SHA-256 HMAC validation prevents unsigned callers from impersonating GitHub.

No application wallet key

KeeperHub owns the managed execution boundary; MergePay stores no blockchain private key.

Replay resistance

GitHub delivery IDs become KeeperHub organization-scoped idempotency keys.

Fail closed

No confirmed hash means no completed payout record.

KeeperHub integration

MergePay uses KeeperHub because the sponsor turns an authorized agent decision into a dependable blockchain transaction. Every live payout is simulated first, executed through the Direct Execution API, retried only when safe, and resolved through KeeperHub's status endpoint.

KeeperHub provides wallet execution, gas management, spending caps, idempotency, run status, transaction hashes and explorer links. MergePay supplies the bounty-specific decision logic.

Setup

KEEPERHUB_MODE=live
KEEPERHUB_API_KEY=kh_...
KEEPERHUB_NETWORK=base
KEEPERHUB_TOKEN_ADDRESS=0x8335...2913
MERGEPAY_WEBHOOK_SECRET=...
MERGEPAY_ADMIN_TOKEN=...
BLOB_READ_WRITE_TOKEN=...

Configure the GitHub webhook at /api/webhooks/github, select pull-request events, and use the same webhook secret in GitHub and Vercel.

Operations

Check /api/readiness before live testing. Use the execution audit to inspect decisions and explorer links. KeeperHub failures are saved as failed records and never represented as paid.
